## 1. Requirements (Context)

- **Goal**: 为 web-nest 增加 i18n 支持，使所有用户可见文本支持中英文切换，跟随系统语言自动检测，允许手动切换
- **In-Scope**:
  - Renderer 所有 UI 文本（WebCatalog、TitleBar、WebAppTitleBar、NavRow、ThemeToggle）
  - Main 进程用户可见文本（tray 菜单、错误对话框、快捷方式描述、错误信息）
  - 语言切换 API（IPC 通信，复用 ServiceRegistry 模式）
  - 翻译文件（en.json、zh-CN.json）
  - 语言偏好持久化
- **Out-of-Scope**:
  - 不支持中文/英文以外的语言（但架构应可扩展）
  - 不翻译日志输出（日志保持英文）
  - 不翻译开发者工具/调试文本
  - 不处理外部 URL 内容的语言

## 2. Research Findings

### 2.1 架构设计（复用 ThemeApi 模式）

**核心思路**：与主题系统完全对齐——主进程是 locale 的 single source of truth，renderer 通过 IPC 获取/切换。

```
┌──────────────────────────────────────────────────┐
│  Main Process                                    │
│  i18next (Node.js)                               │
│  I18nService (Singleton) ← I18nApi (ServiceReg)  │
│  - 检测系统语言 (app.getLocale())                  │
│  - 持久化用户偏好                                 │
│  - 广播语言变更到所有 renderer view                │
├──────────────────────────────────────────────────┤
│  Preload (不变)                                  │
│  IPC 透明传输                                     │
├──────────────────────────────────────────────────┤
│  Renderer Process                                │
│  i18next + react-i18next                         │
│  - 启动时通过 I18nApi.getLocale() 获取语言         │
│  - 监听 IPC 语言变更广播                          │
│  - useTranslation() hook 在组件中使用             │
└──────────────────────────────────────────────────┘
```

### 2.4 关键技术决策

1. **翻译文件位置**：`src/shared/i18n/locales/` — 两端共用，Vite 构建时静态导入 JSON
2. **i18next 初始化**：
   - Main: `i18next.init()` with `initAsync = false`（同步初始化，JSON 资源内联）
   - Renderer: `i18next.use(initReactI18next).init()` 在 `main.tsx` 渲染前完成
3. **语言切换流程**：
   - 用户在 renderer 调用 `i18nApi.setLocale('zh-CN')`
   - Main 的 I18nService 更新 i18next 语言 + 持久化偏好
   - Main 通过 `viewManager.requestTo()` 广播 `locale-changed` 到所有 renderer view
   - Renderer 的 channel listener 更新 i18next 语言
4. **系统语言映射**：`app.getLocale()` 返回值（如 `zh-CN`, `en-US`, `en`）→ 映射到 `zh-CN` | `en`，不匹配时 fallback 到 `en`
5. **持久化**：复用 `~/.web-nest/` 目录，新建 `locale.config` 文件存储用户偏好

### 2.5 风险与不确定项

- **风险**: i18next 作为新依赖，需确保构建产物大小合理
- **风险**: webapp-titlebar 是独立 renderer 入口，也需要初始化 i18next
- **缓解**: 仅引入 `i18next` + `react-i18next`，不用 backend 插件，JSON 静态导入
